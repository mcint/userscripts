# memory/

Notes, not code. The accumulated knowledge that makes the scripts maintainable:

- Per-site quirks — DOM selectors that break, SPA navigation timing, CSP gotchas.
- API notes — MediaWiki Action/REST API endpoints, auth, rate limits.
- Storage keys — which scripts write `localStorage` / cookies / `GM_setValue`,
  and what each key means (so they can be inspected/reset later).
- Decisions — why a script does it one way and not another.

One note per topic; link related notes. This mirrors the seedbed's memory
convention but stays local to this repo.
